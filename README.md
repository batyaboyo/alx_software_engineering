# 💻 ALX Software Engineering Master Monorepo

Welcome to my central software engineering foundations codebase. This repository serves as a unified monorepo houses all 20 foundational tracks, systems architecture, devops scripts, and backend systems completed during my time as a software engineering student at ALX Africa.

Every project included here retains its full atomic tracking history and micro-commit logs, compiled systematically via `git subtree` integrations to showcase continuous development.

---

## 🛠️ Core Technology Stack & Frameworks

* **Languages**: C (Low-Level), Python (High-Level), JavaScript / TypeScript (Backend & Frontend), Bash / Shell (DevOps)
* **Databases & Storage**: MySQL, PostgreSQL, Redis, MongoDB
* **DevOps & Infrastructure**: Nginx, HAProxy, Fabric, Puppet, Docker, Web Server Configuration
* **Testing & Tools**: Unittest, Pytest, Jest, Valgrind, Betty Linter, PEP8/Flake8

## 📦 Consolidated Curriculum Ecosystem

### 📂 1. Core Systems & DevOps Engineering
Low-level scripting, infrastructure configuration, automation pipelines, and Linux server management.
* **`alx-zero_day` / `zero_day` / `alx-pre_course`**: Initial configurations, environment setup, and fundamental Shell navigation.
* **`alx-system_engineering-devops`**: Comprehensive server management tracking including networking, log rotation, automation scripts with Bash, web server optimization (Nginx), and infrastructure monitoring.

### 📂 2. Low-Level Programming & Data Structures (C Language)
Memory allocation, algorithmic optimization, and system-level resource engineering.
* **`alx-low_level_programming`**: Foundational memory manipulation, structures, pointers, hash tables, and custom file handling in standard C.
* **`binary_trees`**: Advanced implementation of binary, AVL, and Max Binary Heap data structures.
* **`sorting_algorithms`**: Time and space complexity profiling for 15+ sorting algorithms (Quick, Merge, Heap, Radix, etc.).
* **`Fix_My_Code_Challenge`**: Debugging project analyzing and resolving hidden logic gaps, segmentation faults, and memory leaks across broken codebases.

### 📂 3. High-Level Programming & Backend Architectures (Python)
Object-Oriented Programming, API design, database tracking, and backend servers.
* **`alx-higher_level_programming`**: Object-Oriented Python, package structuring, SQL database injection prevention, object-relational mapping (SQLAlchemy), and unittest suites.
* **`alx-backend-python` / `alx-backend`**: Core backend mechanics covering asynchronous programming, pagination, user authentication pipelines (Session & Basic Auth), and caching.
* **`alx-backend-storage`**: Database architectures separating File Storage structures from MySQL/NoSQL deployment pipelines.
* **`alx-backend-user-data`**: Design and encryption strategies for handling user metrics, password hashing, and user tracking security.

### 📂 4. JavaScript Stack & Content Management
* **`alx-files_manager`**: A complete backend management file system tracking image manipulation, resizing, background processing via Bull/Redis workers, user tokens, and access verification.
* **`alx-backend-javascript`**: Asynchronous JavaScript, ES6 features, data manipulation, standard API development, and Redis/MongoDB database interactions.

### 📂 5. Advanced Mock Technical Ecosystems
* **`alx-interview`**: A dedicated repository containing daily algorithmic engineering interview problems and optimization strategies (e.g., Pascal's Triangle, Star Pasters, Log Analyzers).
* **`AirBnB_clone` (v1 to v4)**: A 4-stage deployment replicating the core architecture of AirBnB:
  * `AirBnB_clone`: Python-based Command Line Interface (CLI) manipulating static data storage.
  * `AirBnB_clone_v2`: MySQL injection layout integration and storage abstraction.
  * `AirBnB_clone_v3`: RESTful API creation mapping object telemetry datasets to server requests.
  * `AirBnB_clone_v4`: Dynamic web frontend scripting handling real-time data ingestion.
